# MFDO-to-NMDC Biosample Slot Crosswalk (v2)

Maps all 279 Microflora Danica (MFD) habitat ontology leaves to nmdc-schema Biosample slots, covering the ~10,875 environmental biosamples from PRJNA1071982.

**End-to-end result (2025-05-28 db, 10,875 biosamples):**
100% env-triad coverage: `env_broad_scale`, `env_local_scale`, and `env_medium` populated for every sample. GEE-based ELS refinement applied (CORINE preferred over WorldCover). Zero wrong or biome-level ELS CURIEs in output. See [Running the pipeline](#running-the-pipeline).

## Files

### Crosswalk

- **`mfdo_nmdc_crosswalk.tsv`**: primary output (284 rows), sorted by `n_samples` descending so the highest-impact mappings are at the top. A `row_type` column marks each row as `ontology_leaf` (279 genuine MFD leaves) or `biosample_reconciliation` (5 rows added so the join is total over all 10,875 biosamples). An `underspecified_slots` column flags any triad slot left at a coarse root value (bare `biome`, `environmental zone`, or `environmental material`); empty when all three are usable. `terrestrial biome` and other specific biomes are not flagged.
- **`build_ontology_crosswalk.py`**: self-contained, reproducible builder (Python 3 standard library only). Fetches two source xlsx from `cmc-aau/mfd_metadata`; all ENVO term choices and the ELS allow-list are inlined (see [Reproducibility gaps](#reproducibility-gaps)).

### Per-biosample annotation

- **`mfd_biosamples_annotated.tsv`**: the joined per-biosample output, one row per MFD biosample (10,875) with its full NMDC triad. This is the deliverable, and the only view that shows the GEE-refined `env_local_scale` values (the crosswalk alone carries the pre-refinement placeholder). `underspecified_slots` here is recomputed on the post-GEE values. Generated by the two scripts below; committed for review convenience and kept honest by the drift guard.
- **`mfd_db_to_tsv.py`**: fetches the MFD biosample database xlsx from `cmc-aau/mfd_metadata` and extracts the columns `apply_crosswalk.py` expects. Run first. `--help` for options.
- **`apply_crosswalk.py`**: applies the crosswalk and CORINE/WorldCover-based ELS refinement to a biosample TSV, producing `mfd_biosamples_annotated.tsv`. Run `--help` for options. Implements `JOIN_RECIPE.md`.
- **`JOIN_RECIPE.md`**: full documentation of the join recipe that `apply_crosswalk.py` implements.
- **`mfd_gee_landcover.tsv`**: per-coordinate ESA WorldCover + CORINE land cover (pre-computed; no generation script committed yet).

### Land-cover to EnvO mappings

- **`corine_envo_map.tsv`**: all 44 CORINE Level-3 classes mapped to ENVO ELS terms. Entries are `ols_verified=yes` (anchor-class checked), `fails_anchor:*`, or `no anchor-valid ELS`. Used by `apply_crosswalk.py` for GEE ELS refinement.
- **`worldcover_envo_map.tsv`**: all 11 ESA WorldCover 2020 classes. Same convention. Used as fallback when CORINE label is absent (non-EU coordinates).

### Tools (outputs not committed; regenerated on demand)

- **`generate_els_allowlist.py`**: generates the ELS allow-list from ENVO via oaklib and batch-verifies the CURIEs in the mapping TSVs (sets their `ols_verified` column). Writes `els_allowlist.tsv` locally; that file is not committed because it is derived from ENVO in one command and goes stale with each ENVO release. Run with `uv run --extra ontology python3 generate_els_allowlist.py --help`.
- **`search_envo_candidates.py`**: queries OLS4 LLM embeddings (primary) or lexical search (fallback) for ENVO ELS candidates for each row in a land-cover mapping TSV. Works with any TSV that has `*_code` and `*_label` columns. Its candidate output is a curation aid, not committed.

## Running the pipeline

```bash
# 1. Extract biosample TSV from the upstream MFD db xlsx
uv run --extra ontology python3 mfd_db_to_tsv.py --out biosamples.tsv

# 2. Apply crosswalk + GEE ELS refinement
uv run --extra ontology python3 apply_crosswalk.py biosamples.tsv --out annotated.tsv
```

Both scripts accept `--help`. The pipeline produces 100% env-triad coverage over the 2025-05-28 MFD dataset (10,875 biosamples). See `JOIN_RECIPE.md` for the full join recipe and `apply_crosswalk.py` docstring for column details.

## Per-biosample annotation (lookup, not a materialized table)

Annotations are produced by **joining** these small tables to the stock biosample table on demand (see `JOIN_RECIPE.md`): the five MFD levels → this crosswalk (a **total** join — all 10,875 biosamples match exactly one row, including the 116 that match a `biosample_reconciliation` row), and `coord_key` → the GEE tables for optional ELS refinement (only where `coords_reliable == 'Yes'`).

## Reproducing the crosswalk

```bash
python3 build_ontology_crosswalk.py            # default: pinned cmc-aau/mfd_metadata commit
python3 build_ontology_crosswalk.py --ref main # latest upstream release
```

The builder fetches its two source spreadsheets from the public [cmc-aau/mfd_metadata](https://github.com/cmc-aau/mfd_metadata) repo (`data/ontology/latest_mfd-habitat-ontology.xlsx` and `analysis/releases/2025-05-28_mfd_db.xlsx`), pinned to a commit for reproducibility (falls back to same-named files in the working directory if offline). All mapping rules and curated term lists are inlined in the script, so no other inputs are required.

## How mappings are derived

The leaf is identified by the five MFD levels (sampletype, areatype, hab1–hab3), but mappings draw on **all** available leaf-level signal: the five levels and their substrings plus the ontology's `Natura2000`, `EUNIS`, and `EMPO` columns.

- **Natura2000 (Annex I) codes → specific ELS.** Full Annex I habitat names (e.g. 3150 "Natural eutrophic lakes", 7220 "Petrifying springs with tufa formation") are matched to ENVO via an embedding index built over ENVO **labels + definitions + synonyms**, then each chosen CURIE is OLS-verified and confirmed in the ELS allow-list.
- **EMPO → env_medium / salinity** (e.g. saline free water → sea water).
- **Composed-context rules** over the full level hierarchy for specificity; GEE WorldCover/CORINE refinements applied where satellite agreement ≥ 70%.

### Provenance columns

Columns ending in `_provenance` record which input determined each mapping (`from_natura2000`, `from_empo`, `from_mfdo:hab1/2/3`, `from_mfdo:hab3+gee_corine`, `envo_root_class`, …). They are documentation, not nmdc-schema slots, and are stripped before NMDC submission.

### ELS allow-list

Subclasses of (astronomical body part + vegetation layer + environmental zone + terrestrial ecosystem + wetland ecosystem) minus (biomes + materials). Note: `wetland ecosystem` [ENVO:01001209] is a separate root (it is not under `terrestrial ecosystem` in ENVO) to access fen, peatland, sphagnum bog, raised mire, and marsh. See [envo#1659](https://github.com/EnvironmentOntology/envo/issues/1659).

`build_ontology_crosswalk.py` documents 1,533 classes (from the ENVO version used during initial curation); a fresh `generate_els_allowlist.py` run against current ENVO produces ~1,510. The difference is ENVO version. The allow-list is not committed as a file; run `generate_els_allowlist.py` to regenerate it and re-verify the mapping TSVs after an ENVO release.

### ELS fallback

When no specific ELS is supportable from the habitat signal, crosswalk rows use `environmental zone [ENVO:01000408]` as a fallback. This is the root of one of the five ELS anchor classes and is valid per the allow-list. The previous fallback was `astronomical body part [ENVO:01000813]`; it was replaced per NCBI Import Squad decision 2026-06-04.

### geo_loc_name

Not in the crosswalk. All MFD NCBI biosamples carry bare "Denmark". Coordinate-derived locality enrichment is out of scope: it would change a submitter-provided field and is parked pending an NMDC policy decision.

## Data-quality changes in this revision

- **OOXML parse fix.** The reader now places cells by their column reference (Excel omits empty cells from sheet XML); a prior positional parser had shifted hab2/hab3 labels. This corrected the labels and raised biosamples correctly attributed to leaves from 8,696 to **10,759** of 10,875.
- **ELS/EBS/EM refinements** using the multi-system signal above (forest types; grasslands → temperate grassland; humid meadows → wet meadow ecosystem; seabed substrate → sea grass bed / rocky reef; lake trophic state; raised mire; Annex-I-driven eutrophic lake, lagoon, mineral spring, sea cave).
- **Over-assertions removed / realigned** (systematic audit across all leaves). A few leaves were **deliberately backed off to more-general terms** where the prior specific term was unsupported by the inputs — a slight specificity decrease but an accuracy increase:
  - rocky-habitat EBS `temperate shrubland biome` → `terrestrial biome` (23 biosamples)
  - rainwater basins ELS `pond` → `water body` (453)
  - subterranean urban soil ELS `tunnel` → `subsurface landform` (155)
  - non-park urban greenspace ELS `park` → `area of developed open space` (83)
  - petrifying springs EM `peat soil` → `soil` (80; sampletype is Soil, not the deposited tufa)
  - `harbour` is now asserted only where the label says so.

## Reproducibility gaps

Running `python3 build_ontology_crosswalk.py` reproduces `mfdo_nmdc_crosswalk.tsv` from
the two upstream xlsx files with no other inputs. However, three things are embedded in
the script that cannot be regenerated without additional work:

**1. All ENVO term choices for the 279 leaves.** EBS, ELS, and EM values per leaf are
inlined as Python dicts and functions. They were produced by expert curation using
OLS4 embedding search, Natura2000 habitat descriptions, and GEE satellite cross-checks.
Re-deriving them from scratch requires OLS4, runoak + ENVO OBO, and the GEE table.
A future improvement would externalize these into TSV so the curation decisions are
auditable separately from code.

**2. The ELS allow-list (1,533 ENVO classes).** Embedded as a Python set in the script.
Built by running `runoak descendants` against five ENVO anchor classes and subtracting
biomes and materials. `generate_els_allowlist.py` reproduces it on demand from current
ENVO; the script's embedded copy and a fresh run may diverge when ENVO releases a new
version.

**3. The GEE satellite confirmation for four hab3 overrides.** The script hard-codes
four cases where CORINE land-cover satellite data (>=70% agreement) confirmed a specific
ELS choice (Salicornia mud, Spartina swards, Inland dunes Genista, one salt marsh).
Once `corine_envo_map.tsv` coverage is complete, these hard-coded cases should
reference the table instead.

**4. `mfd_gee_landcover.tsv` has no generation script.** The table was produced by a
Google Earth Engine query against the unique coordinates in the MFD biosample set.
If the upstream MFD data adds samples or you need to apply this approach to a different
study, the query must be re-run manually.

## Sources and external dependencies

- **Upstream data:** [cmc-aau/mfd_metadata](https://github.com/cmc-aau/mfd_metadata) supplies
  both the MFD habitat ontology and the per-biosample database xlsx that this crosswalk is built
  from. The build script pins a specific commit for reproducibility.
- **Qualitative pollution values:** the `misc_param` column carries pollution assessments
  (`pollution detected`, `pollution risk`, `pollution level`) for 218 MFD biosamples that encode
  pollution status as a habitat facet. These live in `misc_param` rather than the MIxS `pollutants`
  slot because that slot expects quantitative "name;measurement value" entries and has no home for
  qualitative assessments. Allowing qualitative values there is a proposed nmdc-schema change
  ([microbiomedata/nmdc-schema#3105](https://github.com/microbiomedata/nmdc-schema/issues/3105));
  if accepted, these assertions could migrate from `misc_param` to `pollutants`.
